# Codebase Search Workflow

## Recommended Search Strategy

### Step 1: Understand Project Structure
Start with `get_file_tree()` to see the directory layout:
- Identify key directories (src/, lib/, libs/, packages/)
- Look for configuration files (package.json, binding.gyp, Cargo.toml)
- Find where native code lives (src/, native/, ffi/)

### Step 2: Targeted Keyword Search
Use `search_keyword()` for exact names:
- Function names: `createDecoder`, `init_napi`
- Class names: `FFmpegDecoder`, `NapiHelper`
- File patterns: `binding.gyp`, `*.node`, `addon.cc`
- Import patterns: `require.*native`, `import.*napi`

### Step 3: Semantic Search for Concepts
Use `search_semantic()` when you don't know exact names:
- "how is the native module initialized"
- "audio decoding implementation"
- "NAPI binding configuration"

### Step 4: Scan Results with get_file_info [KEY STEP]
**This is the most useful tool** - rapidly evaluate files without reading full content:
- Use `get_file_info()` for individual files - see functions, classes, imports
- Use `get_files_info()` for bulk scanning - pass all search results at once
- Look for relevant function signatures, class names, import patterns
- Decide which files are worth reading in full

### Step 5: Read Files Strategically
Use `get_file()` ONLY for files that passed the scan:
- Files with relevant functions/classes
- Files with expected import/export patterns
- Configuration files (binding.gyp, package.json)

### Step 6: Narrow with Grep (if needed)
Use `grep_codebase()` only when:
- You need regex patterns like `function.*\(.*env.*\)`
- You suspect index is stale
- You need exact line numbers for editing

## Example: Finding NAPI Integration

1. `get_file_tree({ codebase: "SoundApp" })` → finds libs/native-registry/, libs/ffmpeg-napi-interface/
2. `search_keyword({ codebase: "SoundApp", query: "binding.gyp" })` → finds build configs
3. `get_files_info({ codebase: "SoundApp", paths: [result1.path, result2.path, ...] })` → scan all results
4. `get_file({ codebase: "SoundApp", path: "libs/ffmpeg-napi-interface/binding.gyp" })` → reads selected file
5. `search_keyword({ codebase: "SoundApp", query: "require.*ffmpeg_napi" })` → finds usage locations
6. `get_file_info({ codebase: "SoundApp", path: "js/player.js" })` → scan for functions before reading
7. `get_file({ codebase: "SoundApp", path: "js/player.js" })` → read full file

## Anti-Patterns to Avoid

- Don't start with `search_all_codebases()` - always target a specific codebase first
- Don't use `grep_codebase()` for simple name lookups - `search_keyword()` is 10-30x faster
- **Don't call `get_file()` blindly** - always scan with `get_file_info()` first
- Don't search for vague terms - be specific about what you're looking for
- Don't skip bulk scan - `get_files_info()` on search results saves rounds
